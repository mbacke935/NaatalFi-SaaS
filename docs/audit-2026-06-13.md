## 1. Backend

### 1.1 Faille de sécurité corrigée — suppression d'un admin
`AdminUserDetailView.delete` (`apps/users/views.py`) permettait à un administrateur de **supprimer un autre compte administrateur** sans aucune protection (seule l'auto-suppression était bloquée).

**Correctif** : la suppression d'un compte de rôle `ADMIN` renvoie désormais 400 (« retirez d'abord le rôle admin »). Cohérent avec la protection d'auto-désactivation existante.

Tests ajoutés : `test_admin_cannot_delete_another_admin`, `test_non_admin_cannot_delete_user`.

### 1.2 Durcissement HTTPS production (`check --deploy`)
`manage.py check --deploy` remontait 6 avertissements. Ajout d'un bloc `if not DEBUG:` dans `config/settings.py` :
`SECURE_SSL_REDIRECT`, `SECURE_PROXY_SSL_HEADER` (Render/Vercel terminent le TLS en amont), `SESSION_COOKIE_SECURE`, `CSRF_COOKIE_SECURE`, HSTS 1 an (+ subdomains + preload), `SECURE_CONTENT_TYPE_NOSNIFF`.
Le warning `SECRET_KEY` / `DEBUG` restant disparaît automatiquement en production (vraie clé + `DEBUG=False`).

### 1.3 Couverture de tests complétée
Trous identifiés sur les ajouts récents (EmailLog / cron / Brevo), maintenant couverts :
- Cron : secret **invalide** (≠ absent) → 403 ; **user JWT sans header** → 403 ; **`CRON_SECRET` non configuré** → 403.
- EmailLog : **`queue_email` crée bien un PENDING** ; **email à `max_attempts` n'est plus retraité**.
- Tâches planifiées : **`run_scheduled_tasks` renvoie les 4 tâches OK** ; **`expire_ad_campaigns` expire bien une campagne échue** et préserve les actives.

## 4. UX / UI

Revue visuelle (Edge headless + captures CDP) des pages : accueil, marketplace, fiche produit, login, register, panier, CGU.

### Constats positifs
- Thème sombre cohérent (fond `#0B0B0F`, accent or `#D4AF37`), hiérarchie typographique claire.
- Accueil enrichi : hero, bandeau MVP, carrousel catégories, **produits vedettes avec skeletons**, **section « Boutiques à découvrir »**.
- Fiche produit complète : fil d'ariane, galerie, prix, sélecteur quantité, **bouton favori réactivé**, avis clients, produits similaires.
- Login/CGU : cartes centrées propres, contenu lisible.

### Vérification responsive (point dur élucidé)
Les premières captures `--window-size=375` semblaient montrer un **débordement horizontal** (contenu coupé à droite) sur login / cgu / marketplace.

**Diagnostic rigoureux via le DevTools Protocol** : à toutes les largeurs mesurées, `scrollWidth == innerWidth` (overflow = 0 ; −15px à 478px). Les seuls éléments « hors cadre » sont les cartes du carrousel d'accueil (`home-category-track`), volontairement larges dans un conteneur `overflow-hidden` (marquee) — sans impact sur le `scrollWidth` de la page.

**Conclusion : aucun bug de débordement.** Le « clipping » observé était un **artefact d'Edge headless** : la largeur de viewport minimale (~478 px) est mise en page, mais le canvas de capture demandé (375 px) ne rogne que l'image. Les rendus CDP propres (marketplace, login) confirment un affichage parfaitement contenu. Les correctifs responsive antérieurs (tables `overflow-x-auto`, grilles `gap` adaptées, boutons tactiles) restent valides.

### Points mineurs (non bloquants)
- Images produits en placeholder « IMG » dans l'audit : normal (seed sans upload) ; en production les vraies images Supabase s'affichent.
- Recommandé plus tard : onboarding vendeur guidé après inscription, messages d'erreur encore plus explicites.
